# NPA Agent Roadmap

> 当前阶段：Beta main 增强中
> 原则：先让开源用户稳定跑通样例，再逐步增强外部数据、法律文书、协作和 Skill 生态。

## Alpha 发布前必须完成

目标：别人拿到仓库后，可以本地启动、跑样例、理解边界，并知道哪些数据不会进入 git。

- 确认 README、SECURITY、CONTRIBUTING、发布检查清单和差距盘点一致。
- 跑通 `make setup`、`make test`、`make dev PORT=8767`、`make smoke PORT=8767`。
- 人工走通 5 分钟样例：新建项目、上传 `samples/level1_basic.xlsx`、确认字段、生成报告、生成执行计划。
- 检查 `git status --short --ignored`，确保数据库、上传件、报告、知识库笔记、私有 Skill 草稿正文、`.env` 和真实 API Key 不进入版本库。
- 准备 baseline commit、Alpha tag 和 release notes。

Alpha 可以宣传：

- 本地优先的个贷/消费贷资产包 Excel 初筛。
- 字段识别、数据完整度、画像、处置模式、三情景报价和 Markdown 报告。
- 合同/条款 PDF/DOCX/TXT 的基础文本风险分析。
- 公司历史数据导入、法院画像和报价校准基础版。
- 银登公开 URL/公告文本手动解析。
- DeepSeek、Qwen/百炼、自定义 OpenAI-compatible 的模型调用入口。
- 浏览器语音和 OpenAI-compatible TTS 基础入口。
- 轻量执行工作台、本地 Markdown 知识库、私有 Skill 草稿审核。

Alpha 不应宣传为完整支持：

- 扫描 PDF OCR。
- 判决书、执行文书、调解记录的专门解析。
- AgentMemory MCP。
- BaiLongma 风格完整实时 ASR、流式语音对话和打断播放。
- 银登全站自动监控。
- 外部法院、司法、房产、工商、商业数据源。
- GitHub/community Skill 自动安装和运行。
- 多角色权限、团队协作、生产级 SaaS。
- 完整催收 CRM、真实外呼、短信、企微或自动触达。

## Beta 增强

目标：把 Alpha 的“基础可用”升级为更适合真实处置团队的“材料更全、判断更准、流程更顺”。

- 文书增强：已推进到多模态解析 V2。当前 main 支持 PDF 文本层、图片型 PDF、PNG/JPG/WEBP、DOCX/TXT/HTML、本地可选 OCR、文书解析方式/页码/OCR 状态标注，并把判决/执行/调解线索影响到报价和执行计划。
- 历史校准增强：增加更多匹配维度、样本置信度解释、地区/法院/金额段的可视化对比。
- 银登增强：已启动。当前 main 支持公告附件数量展示、公开 PDF/图片附件解析尝试、机会去重、手动订阅关键词和本地提醒。
- 模型增强：已启动。当前 main 支持用途级模型推荐、prompt 审计摘要和更明确的失败降级动作。
- 语音增强：已启动。当前 main 支持 provider 状态诊断，保留浏览器语音兜底。
- 用户体验增强：统一错误卡、端到端浏览器 smoke、更多样例和导入向导。
- 安全增强：已推进到安全治理与审计工作台 V2。当前 main 支持统一审计 API/前端安全面板、原文云端二次确认、原文敏感导出二次确认、审计摘要和敏感元数据脱敏。

## 后续平台化

目标：在安全可控的前提下，支持团队化、生态化和更多外部数据。

- 外部数据源：法院公开数据、司法类案、执行信息、法拍信息、工商/商业数据源和房产估值。
- 银登自动监控：定时抓取公开机会、去重、摘要、播报和项目创建建议。
- Skill 生态：community/GitHub Skill 的来源校验、版本锁定、权限声明、沙箱运行和禁用机制。
- 私有 Skill 启用：让 approved 草稿经过沙箱、测试和审计后进入分析调用链。
- 多角色权限：管理员、投资分析、法务、执行团队、只读审计等角色。
- 团队协作：项目空间、评论、复盘、分派、导出审批和数据隔离。
- 处置 CRM：案件队列、电话/短信/企微集成、质检、承诺还款跟踪和回款归因。

## 长期原则

- 默认本地优先，敏感数据默认脱敏。
- 云模型和外部数据调用必须说明来源、用途、风险和是否发送敏感信息。
- 法律分析只做辅助提示，不替代律师意见。
- 报价分析只做辅助判断，不构成投资建议。
- 任何 community Skill 或外部自动化能力都必须先有权限声明、审计和可禁用机制。
