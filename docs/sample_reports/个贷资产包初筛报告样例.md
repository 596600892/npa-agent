## 一、结论摘要

**推荐等级：B**

**建议动作：进入初步尽调，优先做电话调解分层，同时对高金额户和深圳管辖子包做重点核验。**

核心机会：

1. 手机号覆盖率较高，12 户中 11 户有手机号，具备电话调解优先的基础。
2. 户均本金约 27,992 元，金额结构适合“电话调解 + 重点户攻坚”的组合处置。
3. 约定管辖法院字段完整，深圳市南山区人民法院占 50%，可考虑对深圳子包评估批量诉讼可行性。
4. 身份证覆盖率较高，11 户可解析年龄和性别，便于做人群画像和地区线索分析。

核心风险：

1. 法院并非高度集中，前 3 个法院覆盖率约 66.7%，未达到批量诉讼默认阈值 70%。
2. 当前未提供合同原文，无法核验管辖条款、送达条款、债权转让通知和诉讼时效。
3. 未上传公司历史处置数据，报价只能采用规则区间，无法做公司专属回收率校准。
4. 存在缺手机号、缺身份证、缺地址的单户，需在处置前补充触达信息。

下一步：

1. 补充合同样本，重点核验深圳子包的管辖、送达、诉讼时效和债权转让通知。
2. 对单户本金前 5% 的高金额户建立重点户清单。
3. 上传公司历史个贷/消费贷处置数据，用于校准报价区间和法院经验。

## 六、处置模式建议

推荐主路径：

```text
电话调解优先 + 高金额户攻坚 + 深圳子包诉讼可行性核验
```

分层策略：

| 层级 | 规则 | 建议 |
|---|---|---|
| A 类 | 手机号完整、金额 5,000-30,000 | 第一轮电话调解，核验还款意愿 |
| B 类 | 金额 30,000 以上且资料完整 | 电话核验后转重点户攻坚 |
| C 类 | 金额低、资料完整 | 批量触达，可设计一次性结清折扣 |
| D 类 | 缺手机号/身份证/地址 | 先补触达信息，不投入高成本处置 |

不建议：

- 不建议一开始全包诉讼，因为法院分散且缺合同原文核验。
- 不建议直接给高报价，因为未接入公司历史处置数据和法院外部数据。

## 七、电话调解话术样例

第一轮核验：

```text
您好，我们这边是受相关债权处置流程委托，和您核实一笔历史借款信息。
为了保护您的个人信息，我先确认一下是否方便沟通。我们这次沟通主要是核实债权事实和了解您的还款意愿。
如果您对金额或债权有异议，可以说明具体情况，我们会记录并反馈核查。
```

协商方案：

```text
如果您有一次性结清或分期处理的意愿，我们可以根据您的实际情况登记协商方案。
具体金额和期限需要经过内部审核确认，不会在电话中强迫您立即决定。
```

合规禁区：

- 不威胁。
- 不骚扰。
- 不联系无关第三人泄露债务信息。
- 不冒充法院、公安或其他机关。
- 不承诺未经授权的减免结果。
